# CLAUDE.md — DeepCurriculum

## Project Overview

DeepCurriculum is an interactive university course platform where all content is **pre-generated at development time** — zero runtime LLM calls. Courses are structured as real university quarter/semester courses with lectures, problem sets, concept checks, quizzes, guided multi-week projects, exams, and capstones.

The platform is a 3-part program training students to become quant engineers:

| Course | Code | Length | Focus |
|--------|------|--------|-------|
| Fundamentals of Compute | CS 101 | 20 weeks | Transistors → processors → GPUs → FPGAs → supercomputers. FPGA trading. C/C++ mastery. |
| AI, Machine Learning & Deep Learning | CS 201 | TBD | All of ML/DL at graduate depth. Neural nets from scratch. AI for trading. |
| Production AI Engineering | CS 291 | 10 weeks | RAG, agents, multi-agent systems, deployment, LLMOps. |

**All students are aspiring quants.** Every course includes dedicated quant/trading sections.

| Math | KaTeX (rehype-katex) | Fast LaTeX |
| Syntax | rehype-highlight | Code highlighting |
| Deploy | Vercel | Zero-config |

### C/C++ Execution Strategy

CS 101 requires running C and C++ code in the browser. Options (in order of preference for MVP):

1. **Python equivalents**: For most exercises, provide a Python implementation that demonstrates the same concept. Students read/write C but execute the Python version in Pyodide. The C solution is shown for reference.
2. **Emscripten/WASM**: Compile C/C++ to WASM and run in browser. More complex to set up but enables true C execution.
3. **Server-side compilation**: API route that compiles and runs C in a sandboxed container. Adds latency and server cost.

**For MVP, use approach #1.** Every C/C++ exercise should have BOTH a C solution (shown, educational) AND a Python equivalent (executable in Pyodide, for testing/grading). The exercise UI shows both tabs.

## Content Quality Standards

### Lectures
- 2000-5000 words. Dense, rigorous, graduate-level.
- Real math with LaTeX. Derive formulas.
- Runnable code examples (C/C++ for CS 101, Python for CS 291).
- 2-4 concept checks per lecture.
- Cite real papers and specs by name.
- Reference guided projects: "You'll implement this in Project 2, Milestone 3."
- Tone: brilliant professor. Rigorous, opinionated, never condescending.

### Exercises
- Dual-language (C + Python) for CS 101. Python-only for CS 291.
- Starter + solution + 4-8 test assertions + 2-3 hints.
- Tests always in Python (Pyodide). Available packages: numpy, pandas, math, collections, json, dataclasses.
- Progressive difficulty within each problem set.

### Guided Projects
- Multi-week with clear milestones.
- Each milestone has: description, requirements checklist, starter code, tests, hints.
- Projects are THE differentiator. They should be substantial enough for a portfolio.
- Projects connect to lecture content — lectures say "you'll build this in Project N."

### Quizzes / Exams
- Weekly: 10-12 questions, 20 min. Midterm: 25 questions, 90 min. Final: 22-35 questions, 60-120 min.
- Every question has an explanation that teaches.

## Content Development Workflow

For each course, content is developed in 4 phases:

1. **RESEARCH**: Subagents use web search to gather real specs, papers, and numbers. Saved to `research/` directory.
2. **SCAFFOLD**: Main session creates course.json, project specs, directory structure.
3. **CONTENT**: Subagents write lectures/exercises/quizzes (2 weeks per subagent). Each reads research notes first.
4. **VERIFY**: Main session checks completeness — file existence, word counts, exercise integrity.

This prevents context overflow and ensures accuracy.

## XP and Progression

- Complete lecture: +50 XP
- Pass exercise: +100 XP (+50 first-try bonus)
- Complete project milestone: +200 XP
- Quiz: score × 2 XP
- Midterm/Final: score × 5 XP

Sequential unlock within each course:
- Lectures in order → all done → problem set unlocks → 60%+ → quiz unlocks
- Weeks unlock after previous week's quiz
- Projects unlock based on their startWeek

Courses are independent — students can take them in any order (though CS 101 → CS 201 → CS 291 is recommended).
